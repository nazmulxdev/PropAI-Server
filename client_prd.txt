1. Project Overview
PropAI is an AI-powered real estate platform designed to solve the fundamental problems of property discovery, listing management, and market intelligence in Bangladesh and beyond. The platform brings together property buyers, sellers/agents, and platform administrators under one unified, intelligent system — powered by real AI APIs and modern web engineering.

Problem Statement:
Finding the right property in Bangladesh is a tedious, manual, and time-consuming process.
Sellers write poor, unoptimized property descriptions that fail to attract buyers.
Buyers receive no personalized property suggestions — everything is generic.
Agents and admins have no intelligent data insights, only raw numbers.
There is no AI-powered assistant to answer buyer questions from real property data.

PropAI Solution:
RAG-based AI Chatbot that answers property queries from actual database content.
AI Content Generator that writes professional property descriptions from basic inputs.
AI Data Analyzer that converts dashboard charts into plain-language business insights.
AI Smart Recommendations that suggest properties based on user behavior and preferences.

On success: redirect to /dashboard/{role}/overview.
Error handling: "Invalid credentials" toast on wrong login.

7.3 Session Management
The session is stored as an httpOnly cookie (better-auth.session_token).
All API calls from Next.js Server Components to the Express backend must forward the cookie header manually:
ts
// Example: Next.js Route Handler calling Express
const sessionToken = cookies().get('better-auth.session_token')?.value;
const res = await fetch(`${EXPRESS_API_URL}/api/properties/saved`, {
  headers: {
    Cookie: `better-auth.session_token=${sessionToken}`,
  },
});
On the client side, Axios is configured with withCredentials: true so cookies are automatically sent to Express.
The Express authenticate middleware reads the cookie (or Authorization header) and validates the session via Prisma against the session table.
No JWTs, no refresh tokens.

7.4 Forgot Password Flow
Step 1: Enter email → Better Auth sends a reset link (via emailAndPassword.sendResetPassword).
Step 2: User clicks link → redirected to /auth/reset-password?token=xxx.
Step 3: New password + confirm → call authClient.resetPassword({ token, newPassword }). On success, redirect to login.
7.5 Protected Routes
All dashboard pages are protected using a middleware that checks the session via auth.api.getSession() from better-auth/next-js in server components, or using useSession() in client components.
If no session, redirect to /auth/login.
Role-based access: if session.user.role doesn't match the required role, redirect to appropriate dashboard or show 403.

11. AI Features — Full Specification
All AI features use real API calls to Anthropic Claude (claude-sonnet-4-20250514) or Google Gemini. No mock/fake responses are acceptable. Every feature must handle loading, error, and success states explicitly.

11.1 AI Content Generator — Property Description & Blog
11.1.1 Property Description Generator
Location: Seller dashboard > Add/Edit Property form > below Description textarea.
Trigger: 'Generate with AI' button with sparkle icon.
How it works:
System collects all filled form data: property type, location, bedrooms, bathrooms, area sqft, amenities, price range, floor.
Sends structured prompt to AI: 'Generate a professional, persuasive property listing description for a {type} in {location}...'
AI returns: description paragraph (200-300 words), 5 key highlights as bullet points, SEO-friendly meta description (150 chars).
Result populates the description textarea and highlights input. User can edit freely after.
UI States:
Default: 'Generate with AI' button, enabled only when at least 3 form fields are filled.
Loading: button shows spinner + 'Generating...' text, button disabled, textarea shows subtle pulse.
Success: textarea fills with content, success toast 'Description generated successfully!', button returns to default.
Error: error toast 'Failed to generate. Please try again.', button returns to default, textarea unchanged.

11.1.2 Blog Post Generator (Admin)
Location: Admin dashboard > Blog Management > Blog Editor.
Trigger: 'AI Write Blog' button.
How it works:
Admin enters blog title and optionally 2-3 keywords.
AI generates: full blog post (600-800 words), intro paragraph, 3-4 subheadings with content, conclusion, 5 SEO tags.
Structured JSON output parsed and populated into editor fields.

11.2 AI Smart Recommendations
Location: Buyer dashboard > AI Recommendations page, and Property detail page 'You might also like' section.
How it works:
Frontend collects user's browsing data: last 10 viewed property IDs, saved property IDs, search query history, location preferences from profile.
Sends this context to backend recommendation endpoint.
Backend calls AI API with: user history summary + catalog of available properties (limited to 50 most recent).
AI returns structured JSON: array of 8 recommended property IDs with reasoning strings.
Frontend fetches full property data for those IDs and displays as cards.
UI States:
Loading: 8 skeleton cards in grid.
Success: 8 property cards with 'Why recommended' tooltip on hover.
Error: error card with retry button.
No history: prompt card encouraging user to browse more properties.

11.3 AI Data Analyzer
Location: Seller Analytics page, Admin Analytics page.
How it works:
After charts render, a 'Get AI Insights' button is visible.

AI returns JSON array of insight objects: { title: string, insight: string, type: 'positive' | 'warning' | 'neutral' }.
Frontend renders insight cards with color-coding: green for positive, orange for warning, blue for neutral.

11.4 RAG AI Chatbot
Location: Floating chat widget, bottom-right corner — available on ALL pages.
What is RAG? Retrieval-Augmented Generation: before responding, the AI retrieves relevant property data from the database, then uses it to answer the question accurately.
How it works:
The user opens a chat widget (click floating button).
The user types a question, e.g. 'Show me 2 bedroom apartments in Dhaka under 20,000 BDT.'
Frontend sends a message to the backend /api/chat/message endpoint.
Backend: (1) extracts keywords and intent from user message, (2) queries PostgreSQL for matching properties, (3) formats top 5 matching properties as context, (4) sends to AI API with system prompt: 'You are PropAI assistant. Answer only using the provided property data context. Be helpful and specific.'
AI responds with natural language answers referencing real properties.
Frontend renders response with message bubbles.
Conversation context maintained for multi-turn: user can ask follow-up questions.
UI Features:
Floating button: PropAI logo icon, pulse animation on first visit.
Chat window: 360px wide, 480px tall. Header with title and close button.
Message bubbles: user messages right-aligned (primary color), AI messages left-aligned (surface color).
Typing indicator: animated dots while AI is responding.
Quick suggestion chips on empty state: 'Apartments in Dhaka', 'Properties under 50 lakh', 'Houses for rent in Chittagong'.
Error: 'Sorry, I could not process that. Please try again.' message bubble.

11.5 AI Auto-Tagging (Bonus Feature)
Location: Seller > Add Property form > Tags input field.
How it works: user clicks 'Auto-Tag' button. AI reads the title and description and returns 5-8 relevant keyword tags as a JSON array. Tags are added to the tags input field automatically.


12. Real-Time Features
12.1 Real-Time Messaging (Socket.io)
Technology: Socket.io client on frontend, Socket.io server on backend.
Connection: established on login, disconnected on logout.
Rooms: each conversation has a unique room ID.
Events emitted: message:send (user sends message), message:read (user reads message).
Events received: message:receive (new message), user:typing (typing indicator), notification:new (new notification).
Messages persist in PostgreSQL. On conversation open, last 50 messages loaded via REST API, then real-time takes over.

12.2 Live Notifications
Bell icon in dashboard navbar with unread count badge.
Dropdown panel (max 10 recent notifications) on bell click.
'View All' link navigates to /dashboard/{role}/notifications.
Notification types: New inquiry received (seller), Seller replied to inquiry (buyer), Listing approved by admin (seller), New message received (both), Property matching preferences found (buyer).
Real-time delivery via Socket.io room event notification:new.
Unread notifications shown with blue dot. 'Mark all as read' button.
